## How to publish on GitHub Pages — Step by step

### Step 1: Create your GitHub account
1. Go to github.com and sign up (free)
2. Choose username: `rajnikaushik` (this becomes your URL)
3. Verify your email

### Step 2: Create your repository
1. Click the **+** icon → "New repository"
2. Name it exactly: `rajnikaushik.github.io`
3. Set to **Public**
4. Tick "Add a README file"
5. Click "Create repository"

### Step 3: Upload your files
1. In your repository, click **"Add file"** → **"Upload files"**
2. Drag and drop ALL files: `index.html`, `style.css`
3. Create the `images` folder: click "Add file" → "Create new file" → type `images/placeholder.txt` → commit
4. Then upload all 8 photos into the images folder

### Step 4: Enable GitHub Pages
1. Go to **Settings** (top menu of your repository)
2. Click **Pages** in the left sidebar
3. Under "Branch" → select `main` → click **Save**
4. Wait 60 seconds → your site is live at `https://rajnikaushik.github.io`

### Step 5: Personalise
- In `index.html`, update your real email address (search for `rajni.kaushik@email.com`)
- Update your LinkedIn and ResearchGate URLs (search for `linkedin.com/in/rajnikaushik`)
- Add more publications to the list as needed

## To update the site later
Just edit the files directly on GitHub (click any file → pencil icon) and commit. Your site auto-rebuilds within 60 seconds.
